# AI 모델 면접 지식정리

> 2026년 3월 기준 업데이트. 각 개념을 **2~3분 내**로 설명할 수 있도록 **"왜 → 무엇을 → 어떻게 → 어디에"** 순서로 구성.

---

## 1. Transformer

```
【Transformer 핵심 개념】

■ 등장 배경
- 2017년 Google "Attention is All You Need" 논문에서 제안
- 기존 RNN/LSTM의 순차처리 한계를 극복하기 위해 설계
- 병렬처리가 가능하여 학습 속도가 획기적으로 향상

■ 핵심 구조
1. Self-Attention Mechanism (자기주의 메커니즘)
   - 입력 시퀀스의 모든 위치 간 관계를 동시에 계산
   - Query, Key, Value 세 벡터로 변환하여 유사도 계산
   - Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V
   - 각 단어가 문장 내 다른 모든 단어와의 연관성을 파악

2. Multi-Head Attention
   - 여러 개의 Attention을 병렬로 수행
   - 다양한 관점에서 문맥 정보를 포착
   - 일반적으로 8개 또는 12개의 Head 사용

3. Positional Encoding
   - 순차처리를 하지 않기 때문에 위치정보를 별도로 주입
   - Sin/Cos 함수를 사용한 고정 패턴 또는 학습 가능한 임베딩
   - 최신: RoPE (Rotary Position Embedding) — 상대적 위치 인코딩, 긴 시퀀스에 유리

4. Encoder-Decoder 구조
   - Encoder: 입력을 고차원 표현으로 변환 (6층 스택)
   - Decoder: 인코딩된 정보를 바탕으로 출력 생성
   - 각 층은 Self-Attention + Feed Forward로 구성

■ 2026년 최신 발전 사항

1. Flash Attention 3 (2024~)
   - NVIDIA Hopper 아키텍처(H100) 최적화
   - warp specialization으로 연산과 데이터 이동 오버랩
   - 원본 대비 2~4× 속도 향상, 긴 시퀀스에서 효과 극대화

2. KV Cache 최적화
   - MHA (Multi-Head Attention): 모든 헤드가 독립 KV — 메모리 부담 큼
   - MQA (Multi-Query Attention): 모든 쿼리 헤드가 하나의 KV 공유 — 최대 절감
   - GQA (Grouped-Query Attention): 쿼리 헤드를 그룹으로 나눠 KV 공유 — MHA 품질 + MQA 효율의 균형
   - MHA → GQA 전환 시 원본 학습량의 일부만으로 uptrain 가능

3. 라우터 기반 동적 어텐션
   - 토큰별로 MHA/GQA/MQA 분기를 동적 선택하는 연구 진행 중
   - 품질과 효율의 최적 균형을 학습으로 탐색

■ 주요 특징 및 장점
- 장거리 의존성(Long-range dependency) 문제 해결
- 병렬처리로 학습 시간 단축
- 문맥을 전역적으로 이해 가능
- Gradient Vanishing 문제 완화 (Residual Connection + LayerNorm)

■ 응용 분야
- NLP: BERT, GPT, T5, Llama, Qwen 등의 기반 모델
- Vision: Vision Transformer (ViT), DiT
- Multi-modal: CLIP, DALL-E, GPT-4V, Gemini
- 번역, 요약, 질의응답, 코드 생성 등 다양한 태스크
```

## 2. VAE (Variational AutoEncoder)

```
【VAE 핵심 개념】

■ 기본 개념
- AutoEncoder의 확률론적 변형 모델
- 데이터를 잠재공간(Latent Space)으로 압축하고 복원하는 생성모델
- 입력 데이터의 확률분포를 학습하여 새로운 샘플 생성 가능

■ 구조
1. Encoder (Recognition Model)
   - 입력 데이터 x를 잠재변수 z의 분포 파라미터로 변환
   - 평균(μ)과 분산(σ²)을 출력
   - q(z|x)를 근사 (사후확률 근사)

2. Latent Space (잠재공간)
   - 일반적으로 정규분포 N(μ, σ²)를 따르도록 제약
   - Reparameterization Trick 사용: z = μ + σ * ε (ε~N(0,1))
   - 이를 통해 역전파 가능

3. Decoder (Generative Model)
   - 잠재변수 z로부터 원본 데이터 x를 복원
   - p(x|z)를 모델링

■ 손실 함수
1. Reconstruction Loss
   - 입력과 복원된 출력 간의 차이
   - MSE 또는 Binary Cross-Entropy 사용

2. KL Divergence
   - 학습된 분포 q(z|x)와 사전분포 p(z) 간의 차이
   - 잠재공간이 정규분포를 따르도록 정규화
   - 과적합 방지 및 연속적인 잠재공간 형성

Total Loss = Reconstruction Loss + β * KL Divergence

■ 특징 및 장점
- 일반 AutoEncoder와 달리 확률적 샘플링으로 새로운 데이터 생성 가능
- 잠재공간이 연속적이고 의미있게 구조화됨
- Interpolation이 가능 (두 데이터 사이의 중간값 생성)
- 이론적으로 견고한 확률론적 프레임워크

■ 단점 및 한계
- 생성 품질이 GAN보다 다소 떨어질 수 있음 (흐릿한 이미지)
- KL Divergence와 Reconstruction 간의 균형 조절 필요
- Posterior Collapse 문제 발생 가능

■ 2026년 현재 위상
- Stable Diffusion의 핵심 구성요소 — 이미지를 Latent Space로 압축하는 역할
- VQ-VAE 계열이 이산(discrete) 표현 학습에 활발히 사용
- Diffusion의 잠재공간 인코더/디코더로서 여전히 필수

■ 응용 분야
- 이미지 생성 및 변환
- 이상탐지 (Anomaly Detection)
- 데이터 압축 및 차원축소
- 약물 설계, 분자구조 생성
- Latent Diffusion의 압축 백본
```

## 3. Diffusion Model

```
【Diffusion Model 핵심 개념】

■ 기본 원리
- 열역학의 확산(Diffusion) 과정에서 영감을 받은 생성모델
- 데이터에 점진적으로 노이즈를 추가하는 Forward Process
- 노이즈에서 원본을 복원하는 Reverse Process를 학습

■ Forward Diffusion Process (순방향 과정)
- 원본 이미지 x₀에서 시작
- T 스텝에 걸쳐 점진적으로 가우시안 노이즈 추가
- 각 스텝마다 q(xₜ|xₜ₋₁) = N(√(1-βₜ)·xₜ₋₁, βₜI)
- 최종적으로 xₜ는 순수한 가우시안 노이즈가 됨
- Markov Chain 구조로 각 스텝은 이전 스텝에만 의존

■ Reverse Diffusion Process (역방향 과정)
- 노이즈 xₜ에서 시작하여 원본 x₀로 복원
- 신경망이 p(xₜ₋₁|xₜ)를 학습
- 실제로는 각 스텝의 노이즈를 예측하도록 학습
- U-Net 기반 아키텍처가 주로 사용됨 → 최근 DiT로 전환 추세

■ 학습 방식
- 목표: 추가된 노이즈를 예측하는 Denoising Network 학습
- 랜덤한 타임스텝 t를 선택하고 노이즈 추가
- 네트워크가 추가된 노이즈 ε를 예측
- 실제 노이즈와 예측 노이즈 간의 MSE Loss 최소화

Loss = E[||ε - ε_θ(xₜ, t)||²]

■ 주요 발전 및 변형
1. DDPM (Denoising Diffusion Probabilistic Models)
   - 기본 Diffusion 모델

2. DDIM (Denoising Diffusion Implicit Models)
   - 샘플링 속도 개선 (스텝 수 감소)
   - Deterministic 샘플링 가능

3. Latent Diffusion Models (LDM)
   - Stable Diffusion의 기반
   - VAE로 압축된 잠재공간에서 Diffusion 수행
   - 계산 비용 대폭 감소

4. Conditional Diffusion
   - Text-to-Image (텍스트 조건)
   - Classifier-Free Guidance로 제어 강화

■ 2026년 최신 발전 사항

5. DiT (Diffusion Transformer)
   - U-Net 대신 Transformer를 백본으로 사용
   - Flux.1-Dev: 12B 생성기 + 4.7B 텍스트 인코더 (CLIP + T5-XXL)
   - 스케일링 법칙이 Transformer와 동일하게 적용됨

6. Flow Matching
   - ODE 기반으로 노이즈→데이터 직선 경로를 학습
   - Diffusion보다 이론적으로 깔끔하고 학습 효율적
   - Stable Diffusion 3, Flux 등에서 채택

7. Rectified Flow
   - 직선(straight) 궤적을 유도하여 추론 스텝 10~100× 감소
   - 음성, 이미지, 비디오, 오디오 편집 등 다영역 채택
   - Variational Rectified Flow Matching (VRFM) — 다중 모드 속도 분포 모델링

8. Consistency Models
   - OpenAI 제안, 1~2스텝 만에 고품질 생성
   - Diffusion의 속도 한계를 근본적으로 해결

■ 단점
- 여전히 GAN 대비 생성 속도 느림 (개선 중)
- 학습에 많은 계산자원 필요
- 정밀한 제어가 어려울 수 있음

■ 응용 분야
- Text-to-Image: Stable Diffusion 3, DALL-E 3, Imagen 3, Flux
- Text-to-Video: Sora, Runway Gen-3, Kling
- Image Editing, Inpainting, Super Resolution
- Audio / Music Generation
- 3D Shape Generation
```

## 4. LoRA / RoLA (Low-Rank Adaptation)

```
【LoRA / RoLA 핵심 개념】

■ LoRA 배경 및 등장
- 대규모 모델의 Full Fine-tuning은 비용이 막대
- 원본 가중치 W를 고정하고 저랭크 분해 행렬만 추가 학습
- W + ΔW, 여기서 ΔW = BA (B: m×r, A: r×n, r<<min(m,n))
- 학습 파라미터 수를 크게 줄임 (0.1~1% 수준)

■ RoLA의 핵심 아이디어 (Rank-One)
- LoRA의 랭크 r을 1로 제한
- ΔW = b·aᵀ (b: m×1, a: n×1)
- 두 개의 벡터만으로 가중치 업데이트 표현
- 극도로 압축된 형태의 적응

■ 구조적 특징
1. 파라미터 효율성
   - LoRA (rank r): 학습 파라미터 = (m+n)×r
   - RoLA (rank 1): 학습 파라미터 = m+n
   - 훨씬 적은 메모리 사용

2. 업데이트 방식
   - 각 레이어마다 학습 가능한 행렬(또는 벡터)만 학습
   - W_new = W_frozen + α·(B·A) 또는 α·(b·aᵀ)
   - α는 업데이트 강도를 조절하는 하이퍼파라미터

■ LoRA 변형 기법 비교 (2026년 기준)
1. LoRA: 가장 기본, rank r 사용
2. RoLA: Rank-1로 극한 압축
3. AdaLoRA: 동적으로 rank 조절 — 중요한 레이어에 높은 rank
4. QLoRA: 4-bit Quantization과 결합 — 단일 GPU로 70B 모델 학습 가능
5. DoRA: 방향과 크기를 분리하여 학습 — Full FT에 더 근접한 성능
6. LoRA+: A와 B 행렬에 서로 다른 학습률 적용

■ 장점
- 극도로 낮은 메모리 사용량
- 빠른 학습 속도
- 모델 병합(Merge)이 용이
- 여러 태스크를 동시에 학습 가능 (Multi-task)
- 원본 모델 구조 변경 없음

■ 단점 및 한계
- 표현력이 Full Fine-tuning보다 제한적
- Rank 선택이 성능에 민감 (보통 4~64)
- 복잡한 태스크에서는 성능 저하 가능

■ 적용 대상
- Transformer의 Attention 가중치 (Q, K, V, O)
- Feed-Forward 레이어
- 대규모 언어모델 (LLM) Fine-tuning
- Stable Diffusion 등 Vision 모델 스타일 적응

■ 실무 활용 팁
- 간단한 도메인 적응: LoRA rank 4~8
- 복잡한 태스크: rank 16~64, DoRA 고려
- 메모리 극한 환경: QLoRA + 4-bit
- 다중 사용자 서빙: LoRA 어댑터를 핫스왑하여 하나의 기본 모델로 다수 서빙
```

## 5. MoE (Mixture of Experts) ⭐ NEW

```
【MoE 핵심 개념】

■ 기본 원리
- 하나의 모델 안에 여러 "전문가(Expert)" 서브네트워크를 두고
- 입력마다 라우터(Router)가 가장 적합한 전문가만 활성화
- 전체 파라미터는 크지만, 추론 시에는 일부만 사용 → 효율적 스케일링

■ 구조
1. Expert Networks
   - 각각 독립된 FFN (Feed-Forward Network)
   - 일반적으로 8~128개의 전문가 보유
   - 각 전문가는 특정 입력 패턴에 특화

2. Router (Gating Network)
   - 입력 토큰을 분석하여 상위 k개 전문가 선택 (보통 Top-2)
   - softmax 기반 확률 분배
   - G(x) = softmax(W_g · x) → Top-k 전문가 선택

3. Sparse Activation
   - 100B 파라미터 모델이지만 추론 시 10B만 활성화
   - 추론 비용은 Dense 모델의 1/k 수준

■ 핵심 수식
Output = Σᵢ G(x)ᵢ · Expertᵢ(x)   (i ∈ Top-k)

■ 주요 모델 사례
- Mixtral 8x7B (Mistral): 8개 전문가 중 2개 활성화, 실제 추론은 ~13B
- DeepSeek-V3: 256개 전문가, 671B 총 파라미터
- Grok-1 (xAI): 314B 파라미터, 8개 전문가
- GPT-4: MoE 구조로 알려짐 (비공개)

■ 학습 시 주요 과제
1. Load Balancing
   - 특정 전문가에 트래픽 집중되는 문제 방지
   - Auxiliary Loss로 균등 분배 유도

2. Communication Overhead
   - 전문가가 여러 GPU에 분산될 때 통신 비용
   - Expert Parallelism으로 해결

3. 학습 불안정성
   - Dense 모델보다 학습 초기 불안정
   - 라우터 초기화 및 학습률 조정이 중요

■ 장점
- 추론 비용 대비 모델 용량 극대화
- 같은 추론 비용으로 Dense 모델보다 높은 성능
- 전문가 추가로 모델 확장 용이

■ 단점
- 메모리 사용량은 전체 파라미터 기준 (100B 모두 메모리에 필요)
- 학습이 Dense 모델보다 복잡
- 배치 크기가 작을 때 효율 저하

■ Dense vs MoE 비교 (면접 빈출)
- Dense 7B ≈ MoE 8x7B(Top-2) 추론 비용
- 그러나 MoE의 실효 용량은 훨씬 큼
- 트레이드오프: 메모리 vs 연산 vs 품질
```

## 6. RLHF / DPO (Alignment) ⭐ NEW

```
【RLHF & DPO 핵심 개념】

■ 왜 Alignment이 필요한가
- 사전학습된 LLM은 "다음 토큰 예측"만 학습
- 유용하고, 정직하고, 무해한 응답 생성을 위해 별도 정렬 필요
- "인간이 선호하는 응답"을 학습시키는 과정

■ RLHF (Reinforcement Learning from Human Feedback)

3단계 파이프라인:

1단계: SFT (Supervised Fine-Tuning)
   - 고품질 데모 데이터로 지도학습
   - 기본적인 지시 따르기 능력 부여

2단계: Reward Model 학습
   - 인간 평가자가 응답 쌍의 선호도 랭킹 (A > B)
   - 이 데이터로 보상 모델 R(x, y) 학습
   - Bradley-Terry 모델: P(A>B) = σ(R(A) - R(B))

3단계: PPO (Proximal Policy Optimization)
   - 보상 모델을 기반으로 RL 최적화
   - KL Divergence 페널티로 SFT 모델에서 너무 벗어나지 않도록 제약
   - L = E[R(x,y)] - β·KL[π_θ || π_ref]

■ DPO (Direct Preference Optimization) — 2026년 주류

핵심 아이디어:
   - 별도 보상 모델 없이 선호 데이터에서 직접 정책 최적화
   - RLHF의 3단계를 1단계로 압축
   - 수학적으로 RLHF와 동치임을 증명

손실 함수:
   L_DPO = -E[log σ(β · (log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))]
   (y_w: 선호 응답, y_l: 비선호 응답)

DPO 장점:
   - 보상 모델 학습 불필요 → 파이프라인 단순화
   - PPO보다 안정적이고 하이퍼파라미터 덜 민감
   - 계산 비용 대폭 절감
   - 2026년 기준 대부분의 주요 AI 랩에서 채택

■ 추가 변형 (면접 보너스 포인트)
- ORPO: SFT와 DPO를 단일 단계로 통합
- KTO: 선호 쌍 없이 개별 "좋음/나쁨" 라벨만으로 학습
- GRPO (DeepSeek): 그룹 단위 상대 보상으로 PPO 단순화

■ 면접 핵심 비교

| 항목 | RLHF (PPO) | DPO |
|------|-----------|-----|
| 보상 모델 | 필요 | 불필요 |
| 학습 안정성 | 낮음 | 높음 |
| 구현 복잡도 | 높음 | 낮음 |
| 계산 비용 | 높음 | 낮음 |
| 이론적 등가 | 기준 | RLHF와 동치 |
```

## 7. RAG (Retrieval-Augmented Generation) ⭐ NEW

```
【RAG 핵심 개념】

■ 왜 RAG가 필요한가
- LLM의 지식은 학습 데이터에 고정 (cutoff 문제)
- 환각(Hallucination) — 없는 사실을 그럴듯하게 생성
- 도메인 특화 지식을 주입하려면 Fine-tuning은 비용 과다
- RAG: 외부 지식을 검색하여 동적으로 주입 → 비용 효율적

■ 기본 아키텍처

1. Indexing (인덱싱)
   - 문서를 청크(chunk)로 분할
   - 각 청크를 임베딩 벡터로 변환 (e.g., text-embedding-3-large)
   - 벡터 DB에 저장 (Pinecone, Weaviate, Chroma, FAISS 등)

2. Retrieval (검색)
   - 사용자 쿼리를 동일 임베딩 모델로 벡터화
   - 벡터 유사도 검색 (Cosine Similarity / ANN)
   - Top-k 관련 청크 반환

3. Generation (생성)
   - 검색된 청크를 컨텍스트로 LLM 프롬프트에 삽입
   - LLM이 컨텍스트 기반으로 응답 생성
   - 출처 추적 가능 (Citation)

■ RAG vs Fine-tuning (면접 빈출)

| 항목 | RAG | Fine-tuning |
|------|-----|------------|
| 지식 업데이트 | 즉시 (문서 교체) | 재학습 필요 |
| 비용 | 낮음 | 높음 |
| 환각 감소 | 효과적 | 제한적 |
| 도메인 적응 | 좋음 | 매우 좋음 |
| 추론 지연 | 검색 지연 추가 | 없음 |
| 스타일 제어 | 제한적 | 효과적 |

■ 2026년 최신 발전 — RAG 2.0

1. 하이브리드 검색 (Hybrid Search)
   - 키워드 기반 (BM25) + 벡터 기반을 결합
   - 정확한 용어 매칭과 의미 유사도 동시 확보

2. 재귀적 검색 (Recursive Retrieval)
   - 검색 → 요약 → 재검색 반복
   - 복잡한 질문에 대해 점진적으로 정보 수집

3. 멀티모달 RAG
   - 텍스트뿐 아니라 이미지, 테이블, 그래프도 검색 대상
   - 멀티모달 임베딩 모델 활용

4. Agentic RAG
   - 에이전트가 검색 전략을 자율적으로 결정
   - 쿼리 분해, 필터링, 재랭킹을 자동 수행
   - 도구 호출과 결합하여 다단계 추론

■ 핵심 최적화 포인트 (실무 면접)
- 청크 크기: 보통 256~1024 토큰, 태스크에 따라 조정
- 임베딩 모델 선택: 도메인에 맞는 모델 중요
- 리랭킹 (Reranking): Cross-encoder로 검색 결과 재정렬
- 프롬프트 엔지니어링: "주어진 컨텍스트에 기반하여 답하세요"
```
