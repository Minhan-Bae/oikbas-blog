## 1. Transformer

```
【Transformer Core Concepts】

■ Background
- Proposed in Google's 2017 paper "Attention is All You Need"
- Designed to overcome the sequential processing limitations of RNN/LSTM
- Parallel processing enables dramatically faster training

■ Core Architecture
1. Self-Attention Mechanism
   - Simultaneously computes relationships between all positions in the input sequence
   - Transforms inputs into three vectors: Query, Key, Value to compute similarity
   - Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V
   - Each word captures its relevance to every other word in the sentence

2. Multi-Head Attention
   - Runs multiple Attention operations in parallel
   - Captures contextual information from diverse perspectives
   - Typically uses 8 or 12 Heads

3. Positional Encoding
   - Since there is no sequential processing, positional information must be injected separately
   - Fixed patterns using Sin/Cos functions or learnable embeddings
   - Latest: RoPE (Rotary Position Embedding) — relative position encoding, advantageous for long sequences

4. Encoder-Decoder Architecture
   - Encoder: Transforms input into high-dimensional representations (6-layer stack)
   - Decoder: Generates output based on encoded information
   - Each layer consists of Self-Attention + Feed Forward

■ Latest Developments as of 2026

1. Flash Attention 3 (2024~)
   - Optimized for NVIDIA Hopper architecture (H100)
   - Overlaps computation and data movement via warp specialization
   - 2–4× speedup over the original, maximized for long sequences

2. KV Cache Optimization
   - MHA (Multi-Head Attention): Each head has independent KV — high memory overhead
   - MQA (Multi-Query Attention): All query heads share a single KV — maximum savings
   - GQA (Grouped-Query Attention): Query heads grouped to share KV — balances MHA quality with MQA efficiency
   - Transitioning from MHA → GQA requires only a fraction of original training for uptrain

3. Router-Based Dynamic Attention
   - Research in progress on dynamically selecting MHA/GQA/MQA branches per token
   - Explores optimal quality-efficiency balance through learning

■ Key Features and Advantages
- Resolves long-range dependency issues
- Reduces training time through parallel processing
- Enables global contextual understanding
- Mitigates Gradient Vanishing problem (Residual Connection + LayerNorm)

■ Applications
- NLP: Foundation models such as BERT, GPT, T5, Llama, Qwen
- Vision: Vision Transformer (ViT), DiT
- Multi-modal: CLIP, DALL-E, GPT-4V, Gemini
- Translation, summarization, question answering, code generation, and more
```

## 2. VAE (Variational AutoEncoder)

```
【VAE Core Concepts】

■ Basic Concept
- A probabilistic variant of the AutoEncoder
- A generative model that compresses data into a Latent Space and reconstructs it
- Learns the probability distribution of input data to generate new samples

■ Architecture
1. Encoder (Recognition Model)
   - Transforms input data x into distribution parameters of latent variable z
   - Outputs mean (μ) and variance (σ²)
   - Approximates q(z|x) (posterior approximation)

2. Latent Space
   - Typically constrained to follow a normal distribution N(μ, σ²)
   - Uses the Reparameterization Trick: z = μ + σ * ε (ε~N(0,1))
   - This enables backpropagation

3. Decoder (Generative Model)
   - Reconstructs original data x from latent variable z
   - Models p(x|z)

■ Loss Function
1. Reconstruction Loss
   - Difference between input and reconstructed output
   - Uses MSE or Binary Cross-Entropy

2. KL Divergence
   - Difference between the learned distribution q(z|x) and prior p(z)
   - Regularizes the latent space to follow a normal distribution
   - Prevents overfitting and forms a continuous latent space

Total Loss = Reconstruction Loss + β * KL Divergence

■ Features and Advantages
- Unlike standard AutoEncoders, can generate new data through probabilistic sampling
- Latent space is continuous and meaningfully structured
- Interpolation is possible (generating intermediate values between two data points)
- Theoretically robust probabilistic framework

■ Limitations
- Generation quality can be somewhat lower than GANs (blurry images)
- Requires balancing KL Divergence and Reconstruction
- Posterior Collapse can occur

■ Current Status as of 2026
- Core component of Stable Diffusion — compresses images into Latent Space
- VQ-VAE variants are actively used for discrete representation learning
- Still essential as the encoder/decoder for Diffusion latent spaces

■ Applications
- Image generation and transformation
- Anomaly Detection
- Data compression and dimensionality reduction
- Drug design and molecular structure generation
- Compression backbone for Latent Diffusion
```

## 3. Diffusion Model

```
【Diffusion Model Core Concepts】

■ Basic Principle
- A generative model inspired by the thermodynamic diffusion process
- Forward Process: Gradually adds noise to data
- Reverse Process: Learns to restore the original from noise

■ Forward Diffusion Process
- Starts from the original image x₀
- Gradually adds Gaussian noise over T steps
- At each step: q(xₜ|xₜ₋₁) = N(√(1-βₜ)·xₜ₋₁, βₜI)
- Eventually xₜ becomes pure Gaussian noise
- Markov Chain structure where each step depends only on the previous step

■ Reverse Diffusion Process
- Starts from noise xₜ and restores to original x₀
- Neural network learns p(xₜ₋₁|xₜ)
- In practice, trained to predict the noise at each step
- Primarily uses U-Net-based architecture → recent trend shifting to DiT

■ Training Method
- Objective: Train a Denoising Network that predicts the added noise
- Select a random timestep t and add noise
- Network predicts the added noise ε
- Minimize MSE Loss between actual and predicted noise

Loss = E[||ε - ε_θ(xₜ, t)||²]

■ Key Developments and Variants
1. DDPM (Denoising Diffusion Probabilistic Models)
   - The foundational Diffusion model

2. DDIM (Denoising Diffusion Implicit Models)
   - Improved sampling speed (reduced step count)
   - Enables deterministic sampling

3. Latent Diffusion Models (LDM)
   - Foundation of Stable Diffusion
   - Performs Diffusion in VAE-compressed latent space
   - Dramatically reduces computational cost

4. Conditional Diffusion
   - Text-to-Image (text conditioning)
   - Enhanced control via Classifier-Free Guidance

■ Latest Developments as of 2026

5. DiT (Diffusion Transformer)
   - Uses Transformer instead of U-Net as the backbone
   - Flux.1-Dev: 12B generator + 4.7B text encoder (CLIP + T5-XXL)
   - Scaling laws apply identically to Transformers

6. Flow Matching
   - Learns straight-line paths from noise to data based on ODE
   - Theoretically cleaner and more training-efficient than Diffusion
   - Adopted by Stable Diffusion 3, Flux, and others

7. Rectified Flow
   - Induces straight trajectories, reducing inference steps by 10–100×
   - Adopted across multiple domains: speech, image, video, audio editing
   - Variational Rectified Flow Matching (VRFM) — models multi-mode velocity distributions

8. Consistency Models
   - Proposed by OpenAI, generates high-quality outputs in just 1–2 steps
   - Fundamentally addresses Diffusion's speed limitations

■ Disadvantages
- Still slower than GANs for generation (improving)
- Requires significant computational resources for training
- Precise control can be challenging

■ Applications
- Text-to-Image: Stable Diffusion 3, DALL-E 3, Imagen 3, Flux
- Text-to-Video: Sora, Runway Gen-3, Kling
- Image Editing, Inpainting, Super Resolution
- Audio / Music Generation
- 3D Shape Generation
```

## 4. LoRA / RoLA (Low-Rank Adaptation)

```
【LoRA / RoLA Core Concepts】

■ LoRA Background and Motivation
- Full Fine-tuning of large-scale models is prohibitively expensive
- Freezes original weights W and trains only low-rank decomposition matrices
- W + ΔW, where ΔW = BA (B: m×r, A: r×n, r<<min(m,n))
- Dramatically reduces the number of trainable parameters (to 0.1–1%)

■ RoLA Core Idea (Rank-One)
- Restricts LoRA's rank r to 1
- ΔW = b·aᵀ (b: m×1, a: n×1)
- Represents weight updates with just two vectors
- An extremely compressed form of adaptation

■ Structural Characteristics
1. Parameter Efficiency
   - LoRA (rank r): Trainable parameters = (m+n)×r
   - RoLA (rank 1): Trainable parameters = m+n
   - Uses significantly less memory

2. Update Method
   - Only trains learnable matrices (or vectors) per layer
   - W_new = W_frozen + α·(B·A) or α·(b·aᵀ)
   - α is a hyperparameter controlling update strength

■ LoRA Variant Comparison (as of 2026)
1. LoRA: The baseline, uses rank r
2. RoLA: Extreme compression with Rank-1
3. AdaLoRA: Dynamically adjusts rank — higher rank for important layers
4. QLoRA: Combined with 4-bit Quantization — enables 70B model training on a single GPU
5. DoRA: Separates direction and magnitude for training — performance closer to Full FT
6. LoRA+: Applies different learning rates to A and B matrices

■ Advantages
- Extremely low memory usage
- Fast training speed
- Easy model merging
- Multiple tasks can be trained simultaneously (Multi-task)
- No changes to the original model architecture

■ Disadvantages and Limitations
- Expressiveness is more limited than Full Fine-tuning
- Rank selection is performance-sensitive (typically 4–64)
- Performance may degrade on complex tasks

■ Application Targets
- Transformer Attention weights (Q, K, V, O)
- Feed-Forward layers
- Large Language Model (LLM) Fine-tuning
- Style adaptation for Vision models like Stable Diffusion

■ Practical Tips
- Simple domain adaptation: LoRA rank 4–8
- Complex tasks: rank 16–64, consider DoRA
- Memory-constrained environments: QLoRA + 4-bit
- Multi-user serving: Hot-swap LoRA adapters to serve multiple users from a single base model
```

## 5. MoE (Mixture of Experts) ⭐ NEW

```
【MoE Core Concepts】

■ Basic Principle
- Places multiple "Expert" sub-networks within a single model
- A Router activates only the most suitable experts for each input
- Total parameters are large, but only a subset is used during inference → efficient scaling

■ Architecture
1. Expert Networks
   - Each is an independent FFN (Feed-Forward Network)
   - Typically 8–128 experts
   - Each expert specializes in certain input patterns

2. Router (Gating Network)
   - Analyzes input tokens and selects the top-k experts (usually Top-2)
   - Probability-based distribution via softmax
   - G(x) = softmax(W_g · x) → Top-k expert selection

3. Sparse Activation
   - A 100B parameter model activates only 10B during inference
   - Inference cost is 1/k of a Dense model

■ Core Formula
Output = Σᵢ G(x)ᵢ · Expertᵢ(x)   (i ∈ Top-k)

■ Key Model Examples
- Mixtral 8x7B (Mistral): 8 experts, 2 activated, effective inference ~13B
- DeepSeek-V3: 256 experts, 671B total parameters
- Grok-1 (xAI): 314B parameters, 8 experts
- GPT-4: Known to use MoE architecture (not publicly confirmed)

■ Key Training Challenges
1. Load Balancing
   - Preventing traffic concentration on specific experts
   - Auxiliary Loss to encourage even distribution

2. Communication Overhead
   - Communication costs when experts are distributed across multiple GPUs
   - Solved via Expert Parallelism

3. Training Instability
   - Less stable than Dense models in early training
   - Router initialization and learning rate tuning are critical

■ Advantages
- Maximizes model capacity relative to inference cost
- Higher performance than Dense models at the same inference cost
- Easy to scale by adding experts

■ Disadvantages
- Memory usage is based on total parameters (all 100B must be in memory)
- Training is more complex than Dense models
- Efficiency drops with small batch sizes

■ Dense vs MoE Comparison (Frequently Asked in Interviews)
- Dense 7B ≈ MoE 8x7B(Top-2) inference cost
- However, MoE's effective capacity is much larger
- Trade-off: Memory vs Computation vs Quality
```

## 6. RLHF / DPO (Alignment) ⭐ NEW

```
【RLHF & DPO Core Concepts】

■ Why Alignment Is Needed
- Pre-trained LLMs only learn "next token prediction"
- Separate alignment is needed to generate helpful, honest, and harmless responses
- The process of training models to produce "human-preferred responses"

■ RLHF (Reinforcement Learning from Human Feedback)

3-Stage Pipeline:

Stage 1: SFT (Supervised Fine-Tuning)
   - Supervised learning with high-quality demonstration data
   - Provides basic instruction-following capability

Stage 2: Reward Model Training
   - Human evaluators rank response pairs by preference (A > B)
   - Train a reward model R(x, y) from this data
   - Bradley-Terry model: P(A>B) = σ(R(A) - R(B))

Stage 3: PPO (Proximal Policy Optimization)
   - RL optimization based on the reward model
   - KL Divergence penalty constrains deviation from the SFT model
   - L = E[R(x,y)] - β·KL[π_θ || π_ref]

■ DPO (Direct Preference Optimization) — Mainstream in 2026

Core Idea:
   - Directly optimizes policy from preference data without a separate reward model
   - Compresses RLHF's 3 stages into 1
   - Mathematically proven to be equivalent to RLHF

Loss Function:
   L_DPO = -E[log σ(β · (log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))]
   (y_w: preferred response, y_l: dispreferred response)

DPO Advantages:
   - No reward model training needed → simplified pipeline
   - More stable than PPO, less sensitive to hyperparameters
   - Dramatically reduced computational cost
   - Adopted by most major AI labs as of 2026

■ Additional Variants (Interview Bonus Points)
- ORPO: Integrates SFT and DPO into a single stage
- KTO: Learns from individual "good/bad" labels without preference pairs
- GRPO (DeepSeek): Simplifies PPO with group-level relative rewards

■ Key Interview Comparison

| Item | RLHF (PPO) | DPO |
|------|-----------|-----|
| Reward Model | Required | Not required |
| Training Stability | Low | High |
| Implementation Complexity | High | Low |
| Computational Cost | High | Low |
| Theoretical Equivalence | Baseline | Equivalent to RLHF |
```

## 7. RAG (Retrieval-Augmented Generation) ⭐ NEW

```
【RAG Core Concepts】

■ Why RAG Is Needed
- LLM knowledge is fixed to training data (cutoff problem)
- Hallucination — generates plausible but nonexistent facts
- Fine-tuning for domain-specific knowledge injection is too expensive
- RAG: Retrieves external knowledge and injects it dynamically → cost-efficient

■ Basic Architecture

1. Indexing
   - Split documents into chunks
   - Convert each chunk to an embedding vector (e.g., text-embedding-3-large)
   - Store in a vector DB (Pinecone, Weaviate, Chroma, FAISS, etc.)

2. Retrieval
   - Vectorize user query with the same embedding model
   - Vector similarity search (Cosine Similarity / ANN)
   - Return Top-k relevant chunks

3. Generation
   - Insert retrieved chunks as context into the LLM prompt
   - LLM generates a response based on the context
   - Source tracking possible (Citation)

■ RAG vs Fine-tuning (Frequently Asked in Interviews)

| Item | RAG | Fine-tuning |
|------|-----|------------|
| Knowledge Update | Immediate (swap documents) | Retraining required |
| Cost | Low | High |
| Hallucination Reduction | Effective | Limited |
| Domain Adaptation | Good | Very Good |
| Inference Latency | Additional retrieval latency | None |
| Style Control | Limited | Effective |

■ Latest Developments as of 2026 — RAG 2.0

1. Hybrid Search
   - Combines keyword-based (BM25) + vector-based search
   - Achieves both exact term matching and semantic similarity

2. Recursive Retrieval
   - Search → Summarize → Re-search cycle
   - Progressively gathers information for complex questions

3. Multimodal RAG
   - Retrieval targets include images, tables, and graphs, not just text
   - Leverages multimodal embedding models

4. Agentic RAG
   - Agent autonomously determines retrieval strategy
   - Automatically performs query decomposition, filtering, and re-ranking
   - Combines with tool calling for multi-step reasoning

■ Key Optimization Points (Practical Interviews)
- Chunk size: Typically 256–1024 tokens, adjusted per task
- Embedding model selection: Domain-appropriate model is critical
- Reranking: Cross-encoder for re-ordering search results
- Prompt engineering: "Answer based on the given context"
```
